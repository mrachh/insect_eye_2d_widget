\section{chunkie}
\subsection{Option 1: Command line installation}
To get chunkie with all its dependencies, clone it from~\url{https://github.com/fastalgorithms/chunkie}
as \vspace*{2ex}\newline
\ovalbox{git clone --recurse-submodules https://github.com/fastalgorithms/chunkie.git}
\vspace*{2ex}\newline
You will need to switch to the \texttt{dev-rcip} branch to use the widget. This can be done by running
\vspace*{2ex}\newline
\ovalbox{git checkout dev-rcip} 
\vspace*{2ex}\newline
Then you can run \texttt{startup.m} to add both chunkie and its dependencies to the
MATLAB path. 

\subsection{Option 2: Package download and install}
Download the \texttt{dev-rcip} branch of chunkie from~\url{https://github.com/fastalgorithms/chunkie} 
and download FLAM from~\url{https://github.com/klho/FLAM}. In each of the respective directories
run \texttt{startup.m} to add the relevant directories to the MATLAB path.

\subsection{Testing the installation}

In order to test the accurate installation of the correct version of the chunkie package,
you can run \texttt{testclm4.m} located at \texttt{chunkie/clmtest}. 
The code should run in a few seconds,
(it takes less than 5 seconds on a 2019 Macbook Pro) and the last line should report 
an error less than $10^{-12}$.

If you can't find the relevant folder, then with
high likelihood you are on the incorrect branch.


\section{fmm2d}
For some of the post processing, you will also need the matlab interfaces from the fast
multipole library in two dimensions. You can try using the precompiled MATLAB interfaces
available in the \texttt{fmm2d\_matlab\_binaries} folder. To make them available to the widget, 
just run \texttt{startup\_fmm2d.m} to add the relevant folder to the PATH. 

To verify successful installation of the fmm2d library, you can run the {\texttt test\_fmm.m} 
script in the {\texttt folder}. It should report back with \texttt{Successfully completed 2 out of 2 tests in fmm2d testing suite}.

In case the installation fails, you can generate the MATLAB interfaces for the fmm2d library by first getting 
the package from~\url{https://github.com/flatironinstitute/fmm2d}. You will need to copy over the appropriate {\texttt make.inc.$\star$} file, depending
on your operating system and compiler installed to {\texttt make.inc}  
followed by running \texttt{make matlab} and \texttt{matlab/startup\_fmm.m}.  For example on a windows machine, you would run the following from a command prompt
\vspace*{2ex}\newline
\ovalbox{copy make.inc.windows make.inc} \vspace*{1ex}\newline \ovalbox{make matlab} 
\vspace*{2ex}\newline
You would still need to run the \texttt{matlab/startup\_fmm.m} script from MATLAB to set the path.
For detailed installation instructions see~\url{https://fmm2d.readthedocs.io/en/latest}. 
If you are going this route, make sure that the path to the \texttt{fmm2d/matlab} folder is higher in priority
in your MATLAB path than the \texttt{insect\_eye\_2d\_widget/fmm2d\_matlab\_binaries} folder.
